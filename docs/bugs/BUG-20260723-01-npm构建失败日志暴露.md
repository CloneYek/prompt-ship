# BUG-20260723-01 — npm 构建失败日志直接暴露给前端用户

## 现象

Vue 项目 `npm install` 或 `npm build` 失败时，前端聊天页构建卡片下方以 `<pre>` 块直接渲染完整 npm 错误日志原文，包含：

- `npm ERR!` 调用栈
- 服务器文件路径（如 `D:\projects\prompt-ship\tmp\code_output\vue_app_xxx\`）
- 依赖安装细节、node_modules 路径

普通用户不应看到这些技术细节。

## 复现步骤

1. 创建 Vue 工程化应用，触发 AI 生成且依赖安装/构建失败
2. 观察 `AppChatPage.vue` 聊天流中 BUILD 卡片的渲染内容
3. 失败卡片下方出现 `pre.build-log` 块，内容为 `buildResult.message` 原文

## 定位过程

| 环节 | 文件 | 行为 |
|------|------|------|
| 后端构建 | `VueProjectBuilder.java:50` | `ProcessBuilder` 执行 npm，失败时返回包含完整 stderr 的 `BuildResult` |
| SSE 传输 | `AppServiceImpl.bridgeVueTokenStream` | `{"b":"fail","msg":"npm ERR!..."}` 通过 SSE 推送给前端 |
| 前端渲染 | `AppChatPage.vue:72-78` | `v-if="status==='fail' && message"` 条件满足，`<pre>{{ message }}</pre>` 直接渲染 |
| 用户可见 | 浏览器 | 完整 npm 日志原文暴露 |

## 根因

整个链路在设计时未区分"服务端排查用日志"和"用户可见提示"。`BuildResult.message` 同时承担了两种职责，前端无条件渲染。

## 修复方案

仅改前端 `AppChatPage.vue`：

1. **删除** `<pre class="build-log">` 块（原 L72-78），不再渲染 `buildResult.message`
2. **替换** `getBuildCardDescription` 失败文案：`"后端构建时遇到错误，下面展示后端返回的关键信息。"` → `"Vue 项目构建失败，请调整需求后重试，或联系管理员查看日志。"`

后端 `VueProjectBuilder` 的 `log.error` 保留不变，服务端日志继续记录完整 npm 输出供管理员排查。

## 验证结果

- `npm run type-check` — 通过
- `npm run build` — 通过（仅有 Vite chunk size 警告，符合预期）
- 构建失败卡片：用户只看到标题"Vue 项目构建失败" + 友好文案，不再出现 `pre.build-log`
