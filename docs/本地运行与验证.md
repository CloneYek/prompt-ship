# 本地运行与验证

## 1. 后端环境

- JDK：必须使用 JDK 21。
- 服务端口：`8123`
- context-path：`/api`
- API 文档：`http://localhost:8123/api/doc.html`
- 数据库：MySQL，默认本地账号 `root/123456`，数据库 `prompt_ship`

## 2. Maven 命令

```bash
# Maven 编译
JAVA_HOME="C:/Users/cy/.jdks/ms-21.0.10" mvn clean compile

# 启动 Spring Boot
JAVA_HOME="C:/Users/cy/.jdks/ms-21.0.10" mvn spring-boot:run

# 运行单个测试类
JAVA_HOME="C:/Users/cy/.jdks/ms-21.0.10" mvn test -Dtest=ClassName
```

## 3. Nginx 静态部署

```bash
D:/Nginx/nginx-1.28.3/nginx.exe                          # 启动
D:/Nginx/nginx-1.28.3/nginx.exe -s reload                # 重新加载配置
D:/Nginx/nginx-1.28.3/nginx.exe -s quit                  # 停止
D:/Nginx/nginx-1.28.3/nginx.exe -t                       # 校验配置
```

Nginx 监听 80 端口，root 指向 `tmp/code_deploy/`。

## 4. 前端验证

常规验证：

```bash
cd prompt-ship-frontend
npm run type-check
npm run build
```

如果当前 shell 找不到 `npm`，可使用 Codex bundled Node 直接运行本地脚本：

```powershell
& 'C:\Users\cy\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' '.\node_modules\vue-tsc\bin\vue-tsc.js' --build
& 'C:\Users\cy\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe' '.\node_modules\vite\bin\vite.js' build
```

Vite 大 chunk 提示属于构建警告，除非任务专门要求优化包体积。

## 5. Windows 注意事项

- 终端中文乱码不一定代表文件损坏，优先用 UTF-8 读取文件并检查替换字符 `\uFFFD`。
- API 测试包含中文 JSON 时，优先使用 UTF-8 文件配合 `--data-binary @file`。
- Windows 沙箱下若出现 `spawn EPERM`、文件移动权限错误等，先判断是否为沙箱限制，再决定是否请求提权重跑。
