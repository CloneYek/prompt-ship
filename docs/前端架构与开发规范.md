# 前端架构与开发规范

## 1. 技术栈

- Vue 3 + Composition API
- TypeScript
- Vite
- vue-router
- Pinia
- Ant Design Vue
- Axios
- markdown-it + DOMPurify
- @umijs/openapi

## 2. API 分层

```txt
src/api/generated/  OpenAPI 机器生成代码
src/api/services/   手写业务 API 封装
```

规则：

- 页面、store、组件应从 `src/api/services/` 导入接口。
- 不手写修改 `src/api/generated/`，除非用户明确说明这是本次 regenerated 内容。
- generated 函数名和参数形状不稳定时，在 services 中包装成稳定业务函数。
- services 负责统一处理：
  - id 字符串/数字转换
  - Blob 下载
  - SSE 流式读取
  - 前端 VO 归一化
  - 页面不应感知的接口细节

## 3. 页面边界

- `HomeView.vue`：首页创建入口、我的作品、案例广场。我的作品可进入对话页，案例广场应优先打开静态预览。
- `AppChatPage.vue`：应用对话、历史消息、SSE 流式展示、预览、部署、下载。
- `MyAppsPage.vue`：当前用户应用管理，允许进入对话页、重命名、删除。
- 管理页面通过 admin service 和 admin API 管理用户/应用。

## 4. 对话页规则

- 首轮生成走 `/app/chat`，前端不提前固定生成类型。
- 收到 SSE `r` 事件后更新当前生成类型展示和预览路径。
- 续聊走 `/app/chat/continue`，沿用已有应用 `codeGenType`。
- 历史消息统一成前端消息模型后渲染。
- `role=user` 展示纯文本气泡。
- `role=assistant` 支持 STEP 卡片和 Markdown 渲染。
- AI 输出 Markdown 时使用 DOMPurify 清洗，避免 XSS 风险。

## 5. UI 风格

- 沿用现有 Ant Design Vue + scoped CSS。
- 工具类按钮优先使用图标，文本命令保持明确。
- 不为了单个功能引入新的 UI 组件库或全局样式体系。
- 页面内交互应有 loading、disabled、错误提示和空态。

## 6. 验证

```bash
npm run type-check
npm run build
```

若 `npm` 不在 PATH，可参考 `docs/本地运行与验证.md` 使用 bundled Node 直接运行本地 `vue-tsc` 和 Vite。
