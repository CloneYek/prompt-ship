# REQ-20260722-01-文档治理体系改造

## 背景

当前 `AGENTS.md` 同时承载 Agent 行为、项目架构、运行命令、SSE 协议、前端约定等内容，职责较杂。用户希望参考轻量文档治理范式，将需求、方案、进度、缺陷和学习总结持久化，便于后续 AI 对话恢复上下文和长期维护。

## 目标

- 将 `AGENTS.md` 重构为 AI 编码 Agent 的行为入口。
- 在 `docs/` 下建立轻量文档治理体系。
- 明确 `REQ / DESIGN / BUG / PROG / NOTE` 五类文档的用途、命名和关联规则。
- 将项目运行、前端规范、后端规范、AI 生成链路等项目知识拆分到独立文档。

## 范围

- 重写 `AGENTS.md`。
- 新增 `docs/文档治理规范.md`。
- 新增项目上下文文档和五类文档目录。
- 新增当天进度日志。

## 验收标准

- `AGENTS.md` 不再重复项目百科，只保留 Agent 行为、读取顺序、代码边界和验证要求。
- `docs/` 下有清晰的文档分类和命名规则。
- 复杂任务可以按 `REQ -> DESIGN -> PROG` 的方式追溯。
- 新增 Markdown 文件均为 UTF-8，可正常读取。

## 非目标

- 不补齐历史所有功能的需求、方案和缺陷文档。
- 不重写 `README.md`。
- 不修改业务代码。
