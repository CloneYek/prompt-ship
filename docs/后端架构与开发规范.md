# 后端架构与开发规范

## 1. 分层职责

```txt
controller/   接收 HTTP 请求、鉴权、参数入口、SSE 桥接
service/      业务接口
service/impl/ 业务实现
mapper/       数据访问
core/         AI 代码生成核心流程
ai/           LangChain4j 接口、Agent、工具调用
model/        entity、dto、vo、enum
```

## 2. Controller 边界

- Controller 不直接写 SQL。
- Controller 不直接调用第三方数据源。
- Controller 不承载复杂业务逻辑。
- 鉴权使用项目已有 `@AuthCheck` 模式。
- SSE 接口使用 `SseEmitter` 桥接 Service 返回的流式内容。

## 3. Service 边界

- Service 负责业务编排、权限校验、实体转换、调用 AI 生成链路。
- MyBatis-Flex Service 可继承 `ServiceImpl<Mapper, Entity>` 获得 CRUD 能力。
- 查询优先沿用 QueryWrapper 和现有 Lambda 字段引用风格。
- 参数错误使用 `ThrowUtils.throwIf` + `ErrorCode` + `BusinessException`。

## 4. AI 代码生成边界

- 生成代码输出到 `tmp/code_output/{codeGenType}_{appId}/`。
- 部署文件输出到 `tmp/code_deploy/{deployKey}/`。
- 下载源码时只能打包当前应用源码目录，必须过滤 `node_modules`、`.git`、`dist`、日志、临时文件等无关内容。
- 任何文件工具都必须限制在当前 app 目录下，不允许访问其他应用或项目敏感目录。

## 5. 数据库与实体

- 数据库结构变化应先写 SQL/迁移和约束，再改实体、Mapper、Service、Controller。
- VO 面向前端展示，敏感字段必须脱敏。
- `AppVO.isOwner` 用于告诉前端当前登录用户是否为应用创建者，前端据此区分可编辑对话和仅预览场景。

## 6. 验证建议

- 普通后端改动：运行 Maven 编译。
- Service 逻辑复杂：补充或运行相关测试。
- 接口变更：同步 OpenAPI/generated，或明确说明未同步原因。
- SSE/AI 链路变更：必须手动验证流式输出、完成事件、异常事件和历史保存。
