# DESIGN-20260723-01 — App 名称初始化优化

## 关联需求

- `todo.md` 优化项："思考一下如何初始化App名称，目前采取的是硬编码取用户initPrompt的前十二位。"

## 当前现状

`AppServiceImpl.createApp()` 中，当用户未显式传入 `appName` 时，默认名称取 `initPrompt` 前 12 个字符：

```java
CharSequenceUtil.subPre(request.getInitPrompt(), 12)
```

问题：

| 输入 | 结果 | 问题 |
|------|------|------|
| "请帮我做一个咖啡店官网" | "请帮我做一个咖啡店官" | 中文被硬截断，前缀动词无意义 |
| "个人简历网页，包含专业技能、教育背景..." | "个人简历网页，包含专" | 逗号后的细节信息混入名称 |
| "VibeCoding是什么" | "VibeCoding是什" | 疑问句式不适合作为名称 |

## 设计方案

新增 `generateDefaultAppName(String initPrompt)` 私有方法，五步规则链：

```text
输入 → 取首句 → 去前缀动词 → 疑问句转陈述 → 截断 ≤ 18 字 → 兜底 "未命名应用"
```

### 规则细节

| 步骤 | 操作 | 示例 |
|------|------|------|
| 1. 取首句 | 按 `，,。！？\n` 切割取第一段 | "个人简历网页，包含技能" → "个人简历网页" |
| 2. 去前缀 | 长组合优先匹配（请帮我做一个/生成一个/写一个...） | "请帮我做一个咖啡店官网" → "咖啡店官网" |
| 3. 句式转换 | `X是什么` → `X简介`；`X和Y的区别` → `X与Y对比`；`X怎么做` → `X制作指南` | "VibeCoding是什么" → "VibeCoding简介" |
| 4. 截断 | `CharSequenceUtil.subPre(first, 18)` | 超过 18 字才截，短名保留原样 |
| 5. 兜底 | 两步回退：原始首句 → `"未命名应用"` | 空白 prompt 等极端情况 |

### 效果对比

| 输入 | 之前（前 12 字） | 之后 |
|------|-----------------|------|
| 请帮我做一个咖啡店官网 | "请帮我做一个咖啡店官" | "咖啡店官网" |
| 个人简历网页，包含专业技能... | "个人简历网页，包含专" | "个人简历网页" |
| VibeCoding是什么 | "VibeCoding是什" | "VibeCoding简介" |
| React和Vue的区别 | "React和Vue的区" | "React与Vue对比" |

## 风险与取舍

- **不保证 100% 准确**：本地规则无法覆盖所有中文表达。用户觉得名字不好可手动重命名（`updateApp` 不改）。
- **不调 AI 命名**：避免每次创建 App 额外增加 LLM 调用成本和延迟。
- **后续可替换**：方法签名独立，未来可替换为 AI 命名而不影响调用方。
- **not a breaking change**：仅影响新建应用时的默认名，已有应用不受影响。
